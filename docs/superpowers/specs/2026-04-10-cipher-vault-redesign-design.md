# CIPHER VAULT — UI全体リデザイン 設計書

**作成日**: 2026-04-10
**対象**: `apps/web/src/pages/LockScreen.tsx`, `apps/web/src/pages/VaultDashboard.tsx`, `apps/web/src/styles.css`

---

## 概要

パスワードマネージャーのフロントエンド全体を「暗号化・解読」をテーマとした**CIPHER VAULTスタイル**にリデザインする。セキュリティツールとしての文脈をビジュアル言語に変換し、ターミナル/暗号解読の美学でユーザー体験を再構築する。

---

## 1. 基盤設計

### 1.1 カラーパレット

```css
--bg-void:       #000000
--bg-surface:    #080808
--bg-panel:      #0d0d0d
--border:        rgba(0, 255, 255, 0.15)
--border-bright: rgba(0, 255, 255, 0.5)
--text-primary:  #e8f4f4
--text-dim:      rgba(200, 230, 230, 0.5)
--accent-cyan:   #00ffff
--accent-hot:    #00e5ff
--accent-danger: #ff0055
--glyph:         rgba(0, 255, 255, 0.04)
```

### 1.2 タイポグラフィ

- **見出し・ラベル・ボタン**: `"Share Tech Mono"` (Google Fonts)
- **本文・フォーム入力**: `"JetBrains Mono"` (Google Fonts)
- 両フォントはGoogleFonts CDNから読み込む

### 1.3 共通エフェクト

| エフェクト | 実装方法 | 用途 |
|-----------|---------|------|
| スキャンライン | `body::before` 固定overlay, 横縞グラデーション, opacity 0.03 | 全画面 |
| シアングロー | `box-shadow: 0 0 12px rgba(0,255,255,0.3)` | フォーカス・アクティブ要素 |
| 背景バイナリ雨 | `<canvas>` でランダムな0/1文字を描画, opacity 4% | 全ページ背景 |
| コーナー装飾 | `::before/::after` でL字ボーダー | パネル・カード |
| カーソル点滅 | CSS `@keyframes` でブロックカーソルを明滅 | 入力フィールド |

## 2. LockScreen リデザイン

### 2.1 レイアウト

- 現在の左右2カラムレイアウトを廃止
- **全画面センタリング・縦1カラム構造**に変更
- 背景にバイナリ雨Canvasが常時流れる
- 中央に認証パネル（最大幅480px）が浮かぶ

### 2.2 ページロードアニメーション（順番に実行）

1. **0.0–0.3s**: バイナリ雨Canvasがフェードイン
2. **0.3–0.9s**: パネルのボーダーが左上から時計回りに描画
3. **0.9–2.1s**: `CIPHER VAULT` 見出しがランダム文字列から実テキストへグリッチデコード
4. **2.1–2.5s**: フォームフィールドとボタンがフェードイン

### 2.3 パネル構造

```
┌─[ CIPHER VAULT ]─────────────────────────────┐
│                                               │
│  > SYSTEM STATUS: [LOCKED / UNINITIALIZED]    │
│                                               │
│  MASTER KEY _________________________________ │
│              [入力フィールド・下線スタイル]        │
│                                               │
│  [ DECRYPT VAULT ]  または  [ INITIALIZE ]    │
│                                               │
│  ! ERR: エラーメッセージ                        │
└───────────────────────────────────────────────┘
```

### 2.4 UIコンポーネント詳細

- **タイトル**: `Share Tech Mono`、グリッチデコードアニメーション適用
- **状態表示**: `SYSTEM STATUS: LOCKED` / `SYSTEM STATUS: UNINITIALIZED`
- **入力フィールド**: 下線スタイル（`border-bottom`のみ）、フォーカス時にシアングロー、ブロックカーソル点滅
- **ボタン**: `[  DECRYPT VAULT  ]` ブラケットスタイル、ホバーでシアン背景に反転
- **エラー**: `! ERR: ` プレフィックス付きでaccent-danger色

## 3. VaultDashboard リデザイン

### 3.1 レイアウト

2カラム構造を維持（サイドバー + 詳細パネル）。全体をターミナルウィンドウとして再解釈。

```
┌─[ VAULT://SECURE ]──────── ENTRIES:N | UPTIME:MM:SS ── [LOCK] ─┐
│                                                                  │
├──────────────────────┬───────────────────────────────────────────┤
│  > SEARCH___________  │  ┌─[ INSPECTOR ]──────────────────────┐  │
│                       │  │  SERVICE:   github.com             │  │
│  ┌─[ ENTRY ]───────┐  │  │  LOGIN:     user@email.com         │  │
│  │ > github.com    │  │  │  PASSWORD:  ••••••••••  [COPY]     │  │
│  │   user@email    │  │  │  URL:       https://...  [COPY]    │  │
│  │   [仕事][開発]  │  │  │  TAGS:      仕事 / 開発             │  │
│  └─────────────────┘  │  │  NOTES:     ________________       │  │
│                       │  │                                    │  │
│  ┌─[ ENTRY ]───────┐  │  │  [ SAVE CHANGES ]  [ DELETE ]     │  │
│  │   ...           │  │  └────────────────────────────────────┘  │
│  └─────────────────┘  │                                          │
│                       │                                          │
│  [+ NEW ENTRY]        │                                          │
└──────────────────────┴───────────────────────────────────────────┘
```

### 3.2 ヘッダー

- 左: `VAULT://SECURE` — タイプライター風アニメーションで表示
- 右: リアルタイムUPTIMEカウンター（ロック解除からの経過時間）、`[LOCK]`ボタン

### 3.3 サイドバー（エントリ一覧）

- **エントリカード**: ロード時にコンソールprintf風に1行ずつ流れ込む
- **選択状態**: 左端に `>` カーソル出現、シアングロー枠、左ボーダーが太く光る
- **検索フィールド**: `> SEARCH_` のターミナルプロンプト風
- **新規エントリ**: `[+ NEW ENTRY]` ボタンでサイドバー内にコマンド入力風フォームが展開

### 3.4 詳細パネル

- フィールド表示: `KEY:   value` のターミナル出力形式（`JetBrains Mono`）
- 編集フォームはフィールドをインプレースで置き換え
- パスワード欄の`[COPY]`クリック: 一瞬グリッチアニメーション → コピー完了表示
- 削除ボタン: ホバーでaccent-dangerにグロー

### 3.5 サマリーグリッド（3パネル）

廃止せず、ヘッダーのインラインステータスバーに統合。`ENTRIES: N | STATUS: ACTIVE | SELECTED: サービス名`

## 5. 非機能要件

- アクセシビリティ: アニメーションは `prefers-reduced-motion` メディアクエリで無効化可能
- パフォーマンス: Canvas描画は`requestAnimationFrame`使用、アンマウント時にクリーンアップ
- 既存のProps・ロジック・API呼び出しは一切変更しない（スタイルのみ変更）
